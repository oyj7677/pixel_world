# 이미지 픽셀화 공유 샘플 설계

작성일: 2026-05-28  
대상 프로젝트: Pixel World 친구 방

## 목표

방장이 이미지를 업로드하면 브라우저에서 즉시 픽셀화하고, 원본 이미지는 서버에 보내지 않고, 픽셀화된 결과만 방의 참고용 샘플로 공유한다. 방 멤버들은 사이드 패널에서 샘플을 보거나 크게 열어 보며 같은 목표를 협동으로 그릴 수 있다.

## 확정된 제품 방향

- 형식: A. 참고용 샘플 패널
- 모드: 간단 모드
- 맞춤 방식: 전체 넣기만 제공
- 저장 범위: 방 공유 저장
- 등록 권한: 방장만 등록/교체
- 보기 방식: 사이드바 미리보기 + 크게 보기
- 서버 저장: 원본 이미지 없이 픽셀 템플릿만 저장
- 실제 캔버스 반영: 자동 칠하기 없음

## 사용자 흐름

1. 방장이 방 캔버스 오른쪽 샘플 패널에서 이미지를 선택한다.
2. 브라우저가 현재 캔버스 크기에 맞춰 이미지를 전체 넣기 방식으로 축소한다.
3. 투명 영역은 기본 배경색으로 처리하고, 나머지는 Pixel World 팔레트에서 가장 가까운 색으로 양자화한다.
4. 방장은 생성된 픽셀 샘플을 확인한 뒤 방 샘플로 저장한다.
5. 방 멤버는 새로고침 없이 공유 샘플 변경을 확인하고, 필요하면 크게 보기로 연다.
6. 멤버들은 샘플을 참고해 각자 저장된 픽셀을 사용해 실제 캔버스를 칠한다.

## 전체 넣기 정의

전체 넣기는 이미지가 잘리지 않도록 원본 비율을 유지한 채 캔버스 안에 전부 들어오게 맞추는 방식이다. 남는 여백은 캔버스 기본 배경색으로 채운다. 예를 들어 세로 사진을 정사각형 48x48 캔버스에 넣으면 위아래 또는 좌우 여백이 생길 수 있지만, 사진의 중요한 부분이 잘리지 않는다.

꽉 채우기는 반대로 캔버스를 빈틈없이 채우는 방식이라 일부 이미지 가장자리가 잘릴 수 있다. 이번 기능에서는 협동 참고 샘플의 안정성을 위해 꽉 채우기를 제공하지 않는다.

## 데이터 모델

`room_pixel_templates` 테이블을 추가한다.

- `id`: 템플릿 ID
- `room_id`: 방 ID
- `created_by_member_id`: 등록한 방장 멤버 ID
- `name`: 샘플 이름
- `width`, `height`: 캔버스 크기
- `default_color_hex`: 여백/투명 영역 색
- `pixels`: 기본 배경색이 아닌 픽셀 배열 JSON
- `created_at`, `updated_at`, `deleted_at`

방마다 활성 템플릿은 1개만 둔다. 새 템플릿을 저장하면 기존 활성 템플릿은 소프트 삭제하고 새 행을 추가한다.

## API

`GET /api/rooms/:roomPublicId/pixel-template`

- 방 멤버 또는 유효한 초대 인증이 있는 사용자만 조회할 수 있다.
- 응답은 `{ template: RoomPixelTemplateDto | null }` 형태다.

`PUT /api/rooms/:roomPublicId/pixel-template`

- 방장만 저장할 수 있다.
- 요청에는 이름, 크기, 기본 색, 픽셀 배열만 포함한다.
- 서버는 크기, 좌표 범위, 색상 HEX, 픽셀 개수 상한을 검증한다.
- 저장 후 같은 방 소켓에 `roomPixelTemplateUpdated` 이벤트를 브로드캐스트한다.

## 프론트엔드

새 샘플 패널은 기존 정적 샘플 화면을 유지하면서, 방 공유 샘플이 있으면 최상단에 표시한다.

방장에게만 이미지 선택과 저장 버튼을 노출한다. 멤버에게는 등록된 샘플과 크게 보기만 보인다. 업로드된 원본은 브라우저 메모리에서만 사용하며 저장 요청에는 픽셀화 결과만 포함한다.

픽셀화 로직은 UI와 분리해 테스트 가능한 순수 함수로 둔다.

- 원본 이미지 비율 유지
- 중앙 정렬
- 캔버스 밖 잘림 없음
- 알파값이 낮은 픽셀은 기본 배경색 처리
- `DEFAULT_PALETTE`에서 가장 가까운 색상 선택

## 실시간 동기화

템플릿 저장 후 서버가 해당 방의 현재 캔버스 소켓 방에 이벤트를 보낸다. 클라이언트는 이벤트를 받으면 공유 샘플 상태를 갱신한다. 소켓에 연결되어 있지 않은 사용자는 방 진입 시 GET API로 최신 템플릿을 받는다.

## 검증 기준

- 방장은 픽셀 템플릿을 저장할 수 있다.
- 일반 멤버는 저장할 수 없고 403을 받는다.
- 비멤버는 조회/저장할 수 없다.
- 원본 이미지 파일은 서버 요청에 포함되지 않는다.
- 전체 넣기 방식에서 긴 이미지가 잘리지 않는다.
- 공유 샘플이 있는 경우 사이드바와 크게 보기에서 같은 픽셀 결과를 보여준다.
- 저장 후 같은 방에 접속한 클라이언트가 실시간으로 갱신된다.

## 이번 구현 범위

이번 작업에서는 MVP로 다음까지 구현한다.

- DB 마이그레이션
- 공유 DTO와 소켓 이벤트 타입
- 서버 저장/조회 API
- 방장 권한 검증
- 브라우저 픽셀화 유틸
- 방 사이드바 샘플 패널
- 크게 보기
- 서버/API/프론트 유닛 테스트
- 전체 검증 및 수동 QA

다중 샘플, 멤버 제안/방장 승인 큐, 템플릿을 실제 캔버스에 자동 적용하는 기능은 후속 범위로 남긴다.
